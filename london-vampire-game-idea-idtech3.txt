Release: December 1999
Source code: GPL since August 19, 2005
Language: C (some C++ in later forks)
Platform support: Windows, Linux, macOS, various consoles

GRAPHICS

Renderer: OpenGL only. No software renderer.

Surface types: Planar BSP + spline-based curved surfaces.

Shaders: Text file scripts defining surface appearance. Multiple layers with blend modes, environment mapping, scrolling, rotation. Also defines volume properties (water, fog), light emission, footstep sounds.

Lighting: Lightmaps for static geometry. Gouraud shading for models. Colored lights from lightgrid applied to models.

Model format: MD3

Per-vertex animation (not skeletal)

Variable keyframes per second (vs MD2's fixed 10)

Models split into three parts (head/torso/legs) for independent animation blending

Each part has separate texture set

Shadows: Three modes

Blob shadow (circle at feet)

Solid polygonal shadow

Stencil shadow volume (semi-transparent, unclipped—extends through walls)

Effects: Volumetric fog, mirrors, portals, decals, wave-form vertex distortion

Video: RoQ format (vector quantization video + DPCM audio)

SOUND

2-channel output

96 track mixing

Stereo spatialization

Doppler effect

No EAX or surround support

No dedicated sound thread → buffer looping if game stalls

No echoes

NETWORKING

UDP-based snapshot system

Server updates at fixed rate independent of clients

Delta encoding (only differences from last confirmed frame)

Huffman compression on all packets

Pure server mode (file integrity checks, anti-cheat)

VIRTUAL MACHINE

QVM system for server behavior, client effects, UI

LIMITATIONS

No skeletal animation (MD3 is vertex-based, though MD4 partial implementation exists in some forks)

Static lightmaps only (no real-time dynamic lighting)

Shadows unclipped (bleed through geometry)

Sound system primitive (no HRTF, EAX, proper 3D positioning)

Single-threaded

No native controller support (can be added)

BSP compile times long for complex curves

FOR BUILDING YOUR GAME:

Start with ioquake3. Gives you modern backend (SDL/OpenAL) while keeping idTech3 core. Modify:

Custom entity system for detective mechanics

Extended shaders for fog/mood lighting

VM for scripting investigative sequences

Replace MD3 with skeletal system if needed (ioquake3 has some MD3/MD4 hybrid work from Elite Force SDK)
